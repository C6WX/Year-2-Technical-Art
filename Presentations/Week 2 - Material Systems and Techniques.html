What Are Material Functions?

Reusable Graphs Inside Materials

Material Functions are self-contained material graphs that can be reused across multiple materials. Think of them as “functions” in programming - write once, use everywhere.

Why Use Material Functions?

  • Avoid duplicating complex node setups
  • Update all materials at once by editing the function
  • Share functionality across team members
  • Reduce material graph complexity

Material Functions - Tiling Example

Key Concept: Material Functions have inputs and outputs, just like blueprint functions. They encapsulate logic that can be called from any material.

Material Functions - Key Benefits

Modularity & Reusability

  • “One master graph, many children”
  • Store math, UV manipulation, or surface effects
  • Save time and maintain consistency
  • Easy to update and iterate

Common Examples

  • Tiling function
  • Fresnel effects
  • Triplanar blend
  • Custom noise generators
  • UV manipulation utilities
  • Color grading functions

Creating a Material Function

Step-by-Step Process:

  1. Right-click in Content Browser → Materials & Textures → Material Function
  2. Add Function Inputs - Define what parameters your function needs
  3. Build the Logic - Create your node graph
  4. Add Function Output - Define what the function returns
  5. Save and Use - Call it from any material with the MaterialFunctionCall node

Name your inputs clearly and add descriptions. Future you (and your team) will thank you!

Material Parameter Collections (MPC)

Global Parameters Accessible Across Multiple Materials

MPCs are special assets that store parameters accessible by ALL materials in your scene simultaneously. Perfect for scene-wide effects.

How They Work:

  • Create one MPC asset
  • Add scalar and vector parameters
  • Reference from any material
  • Update values at runtime via Blueprint/C++

MPC - When to Use

Perfect For:

  • Time of day systems
  • Weather effects (rain intensity, wetness)
  • Damage systems (global burn amount)
  • UI → world synchronization
  • Global color grading
  • Wind direction and strength

Not Ideal For:

  • Per-object parameters
  • Material-specific settings
  • Instance-level customization

MPC Example: Weather System

Scenario: You want rain to affect all surfaces in your scene.

MPC Parameters:

  • RainIntensity (0-1) - How much it’s raining
  • WetnessAmount (0-1) - How wet surfaces are
  • PuddleDepth (0-1) - Puddle accumulation

Blueprint Control:

MPC - Implementation Details

Creating an MPC:

  1. Content Browser → Right-click → Materials & Textures → Material Parameter Collection
  2. Open the MPC asset
  3. Add Scalar or Vector Parameters
  4. Name them clearly (e.g., Global_SunAngle)

Using in Materials:

  • Add CollectionParameter node (or drag one in from the content drawer)
  • Select your MPC in the details panel
  • Choose the parameter name
  • Connect to your material logic

Dynamic Material Instances & Post Process

Runtime Material Control & Screen Effects

Understanding how to modify materials at runtime and apply camera-based effects for maximum visual impact.

What Are Material Instances?

The Foundation of Runtime Material Control

Material Instances are lightweight copies of a parent material that expose specific parameters for customization.

Two Types:

  1. Material Instance Constant (MIC) - Created in editor, static
  2. Dynamic Material Instance (DMI) - Created at runtime, fully dynamic

Key Difference: MICs are set in the editor and don’t change. DMIs can be modified every frame during gameplay.

Dynamic Material Instances (DMI)

Runtime Material Modification

DMIs are created during gameplay and can have their parameters changed every frame. Essential for interactive and responsive materials.

When to Use DMI:

  • Parameters change during gameplay
  • Need per-object customization
  • Responding to game events
  • Player-driven customization

Important: Create DMI once (e.g., BeginPlay), then reuse it. Don’t create new DMIs every time you want to update a parameter!

DMI - Key Features

Technical Features

  • Created at runtime via Blueprint
  • Change parameters without recompiling
  • Minimal performance overhead
  • Supports all parameter types
  • Can be created from MIC or parent material

Common Applications

  • Health bar color changes
  • Dissolve effects
  • Weapon glow intensity
  • Character customization
  • Interactive UI elements
  • Damage visualization

Creating a DMI - Blueprint Example

Blueprint Setup:

  1. Get the mesh component (Static Mesh, Skeletal Mesh, etc.)
  2. Create Dynamic Material Instance node
    • Source Material: Your material or MIC
    • Element Index: Which material slot (usually 0)
  3. Store the reference in a variable
  4. Use Set Scalar/Vector Parameter Value to modify it

Key Node: Create Dynamic Material Instance - Returns a reference you can modify

DMI Example: Hit detection

Scenario: Character or enemy gets hit and flashes red.

Material Setup:

  • Create material with HitEmissiveColor parameter (Vector3)
  • Create material with HitEmissiveValue parameter (Scalar)

Blueprint Logic:

Alternative Use Case: Character death or teleport effect

Material Parameters:

  • DissolveAmount (0-1) - How much is dissolved
  • EdgeColor - Color of the dissolve edge
  • EdgeWidth - Width of the glowing edge

Material Logic:

  • Use noise texture as dissolve pattern
  • Compare noise with DissolveAmount
  • Clip pixels below threshold
  • Add edge glow where noise ≈ DissolveAmount

Post Process Materials

Available at: https://outsidergaming.com/borderlands-4-unveils-new-trailer-featuring-characters-gameplay-and-a-villainous-twist/ [Accessed 6 Oct. 2025]. ‌

Materials Applied to Camera Pass → Affects Final Image

Post Process materials are special materials that run AFTER the scene is rendered, allowing you to modify the final image before it’s displayed.

How They Work:

  1. Scene renders normally
  2. Post Process material runs on the full-screen image
  3. Material can read scene color, depth, custom depth, etc.
  4. Output becomes the final image

Key Concept: Post Process materials operate on the 2D screen image, not on 3D geometry

Post Process - Blendable Locations

Before Tonemapping

  • Full HDR range available
  • Access to bright values > 1.0
  • Best for bloom, exposure effects
  • Can create physically accurate effects

After Tonemapping

  • LDR (0-1 range)
  • Post-color grading
  • Best for stylized effects
  • Faster performance

Replacing Scene Color:

  • Complete control over final output
  • Can completely override the image
  • Most flexible but most complex

Post Process Material Setup

Creating a Post Process Material:

  1. Create new Material
  2. Set Material Domain to “Post Process”
  3. Set Blendable Location (Before/After Tonemapping)
  4. Connect nodes to Emissive Color output
  5. Add to Post Process Volume or Camera

Post Process materials ONLY use the Emissive Color output. Other outputs are ignored

Post Process Example: Outline Shader

Use Case: Highlight selected objects with colored outline

Material Inputs:

  • Scene Texture: Custom Depth - Objects marked for outline
  • Scene Texture: Scene Color - Original image

Logic:

  1. Sample Custom Depth at current pixel and neighbors
  2. If depth difference detected → edge found
  3. Draw outline color at edges
  4. Blend with original scene color

Post Process Example: Screen Distortion

Use Case: Heat waves, water droplets, damage effects

Material Setup:

  • Sample Scene Color with offset UVs
  • Use noise texture to create distortion pattern
  • Animate noise over time for movement
  • Multiply by intensity parameter

Blueprint Control:

// Increase distortion when player is damaged
PostProcessDMI->SetScalarParameterValue("DistortionIntensity", DamageAmount);

Optimize Performance

  • Keep shader instructions low
  • Avoid complex math
  • Minimize texture samples
  • Use lower resolution when possible
  • Profile with stat RHI

Best Practices

  • Use Post Process Volumes for level-specific effects
  • Blend between effects smoothly
  • Test on target hardware
  • Consider mobile limitations
  • Use Material Quality nodes for scalability

Virtual Textures & Memory

Optimizing Texture Memory Usage

Understanding how to manage texture memory efficiently for large-scale environments and high-resolution assets.

The Texture Memory Problem

Why We Need Virtual Textures

Traditional Texture Loading:

  • Entire texture loaded into VRAM
  • 8K texture = ~256 MB uncompressed
  • Large landscape = dozens of 8K textures
  • Result: VRAM exhausted quickly!

The Solution:

Virtual Textures stream only the visible portions of textures, dramatically reducing memory usage.

A 16K texture would use 1 GB of VRAM if fully loaded. VT can reduce this to 50-100 MB!

Virtual Textures (VT)

Stream Textures in “Tiles” Rather Than Loading Full Texture

Epic Games (n.d.). Streaming Virtual Texturing. [online] https://dev.epicgames.com/documentation/. Available at: https://dev.epicgames.com/documentation/en-us/unreal-engine/streaming-virtual-texturing?application_version=4.27 [Accessed 6 Oct. 2025]. ‌

How VT Works:

  1. Texture divided into small tiles (e.g., 128x128)
  2. Only visible tiles loaded into memory
  3. Tiles stream in/out based on camera view
  4. Automatic mipmap selection per tile

Benefits:

  • Reduce memory usage by 80-90%
  • Essential for huge landscapes
  • Consistent texture quality at any distance
  • Supports textures larger than VRAM

Virtual Texture Setup

Enabling VT in Your Project

Project Settings:

  1. Edit → Project Settings → Engine → Rendering
  2. Enable Virtual Texture Support
  3. Set Virtual Texture Tile Size (128 recommended)
  4. Restart editor

Per-Texture Settings:

  1. Open texture asset
  2. Check Virtual Texture Streaming
  3. Set appropriate Virtual Texture Tile Size
  4. Save and rebuild

Not all textures benefit from VT. Best for large landscape textures and high-res detail maps.

Virtual Texture - When to Use

Perfect For:

  • Landscape base textures
  • Large terrain detail maps
  • Mega-scans assets
  • Open world environments
  • High-resolution detail textures
  • Unique, non-repeating textures

Not Ideal For:

  • Small, frequently used textures
  • UI textures
  • Particle textures
  • Textures < 1024x1024
  • Tiling detail textures
  • Mobile platforms (limited support)

Runtime Virtual Textures (RVT)

Bake Scene Data Into Usable Textures

RVT is a special type of Virtual Texture that captures and stores rendered scene data (color, normals, height) that can be sampled by materials.

Key Difference:

  • VT: Streams pre-authored texture data
  • RVT: Captures and stores runtime scene data

RVT allows materials to “see” what’s rendered above them, enabling effects like footprints, decals, and dynamic blending.

RVT - How It Works

Setup Process:

  1. Place RVT Volume in level (defines capture area)
  2. Configure RVT - Choose what to capture (Base Color, Normal, Height)
  3. Mark objects to render into RVT
  4. Sample RVT in materials using WorldPosition

What Gets Captured:

  • Base Color (RGB)
  • World Normal
  • Height/Displacement
  • Custom data channels

Texture Groups

Organizing Textures by Priority

Texture Groups allow you to set different streaming priorities and quality settings for different types of textures.

Default Groups:

  • World - Environment textures
  • Character - Character textures (high priority)
  • Weapon - Weapon textures (highest priority)
  • Effects - VFX textures (can be lower res)

Custom Groups:

Create groups for your specific needs (e.g., “Hero_Props”, “Background_Clutter”)

Surface Techniques

Advanced Material Surface Methods

Exploring techniques that add depth, detail, and realism to surfaces without adding geometry.

The Depth Illusion Problem

Creating 3D Detail Without Geometry

The Challenge:

  • Flat surfaces look flat
  • Adding geometry is expensive
  • Need depth perception without polygons

Solutions:

  1. Normal Maps - Fake lighting (no actual depth)
  2. BumpOffset - Simple UV shift (cheap parallax)
  3. POM - Advanced raymarching (realistic depth)
  4. Displacement - Actual geometry (expensive)

Goal: Maximum visual depth with minimum performance cost

BumpOffset (Simple Parallax)

Basic UV Offset for Depth Illusion

How It Works:

  1. Sample height map
  2. Offset UVs based on height and view angle
  3. Sample color/normal with offset UVs
  4. Result: Slight depth perception

Material Setup:

Height Map → Multiply by Strength → Add to UV
Offset UV → Sample Base Color

BumpOffset - Pros and Cons

Advantages

  • Very cheap - Single texture sample + math
  • Minimal performance cost - ~0.1ms
  • Works on all platforms - Even mobile
  • No artifacts - Stable at all angles
  • Easy to implement - Simple node setup

Limitations

  • Flat appearance - Limited depth
  • No self-occlusion - Can’t hide surfaces
  • No self-shadowing - No depth shadows
  • Breaks at edges - UV stretching visible
  • Subtle effect - Only works for small details

Parallax Occlusion Mapping (POM)

Advanced Raymarching for Realistic Depth

Fujiwara, H. (2023). Are there really rooms behind every window in Marvel’s Spider-Man? This is how developers create the illusion - AUTOMATON WEST. [online] AUTOMATON WEST. Available at: https://automaton-media.com/en/news/20231201-23558/ [Accessed 6 Oct. 2025]. ‌

How It Works:

  1. Raymarch through height map (8-32 steps)
  2. Find intersection point with surface
  3. Sample textures at intersection
  4. Calculate self-shadowing
  5. Result: Convincing 3D depth

Performance Cost:

  • 8 steps: ~0.5ms
  • 16 steps: ~1.0ms
  • 32 steps: ~2.0ms

POM - Technical Details

Understanding the Raymarching Process

Algorithm:

For each step (0 to NumSteps):
  1. Calculate ray position in height map
  2. Sample height at current position
  3. If ray below surface → intersection found
  4. Refine intersection (optional)
  5. Return final UV offset

Parameters:

  • Height Scale - Depth intensity (0.01-0.1)
  • Min Steps - Quality at perpendicular angles (8-16)
  • Max Steps - Quality at grazing angles (16-32)
  • Refine Steps - Intersection accuracy (0-4)

How it works

https://learnopengl.com/Advanced-Lighting/Parallax-Mapping

POM - Pros and Cons

Advantages

  • Realistic depth - Convincing 3D illusion
  • Self-occlusion - Surfaces hide behind others
  • Self-shadowing - Depth creates shadows
  • Parallax motion - Depth shifts with view
  • Works with lighting - Proper depth response

Limitations

  • Expensive - 8-32 texture samples
  • Grazing angle artifacts - Stretching at edges
  • Silhouette issues - Edges still flat
  • Not suitable for mobile - Too expensive
  • Requires good height maps - Quality dependent

POM - Material Setup

UE5 POM Node:

The ParallaxOcclusionMapping node handles the complexity:

Inputs:

  • Height Map - Grayscale height texture
  • Height Ratio - Depth scale (0.01-0.1)
  • Min Steps - Minimum raymarch steps
  • Max Steps - Maximum raymarch steps

Output:

  • Parallax UVs - Use for all texture samples

ALL textures (color, normal, roughness) should use the POM offset UVs

World Position Offset (WPO)

Vertex Movement in Material Shader

WPO allows you to move vertices in the material shader, creating animation and deformation without skeletal animation or morph targets.

How It Works:

  • Material shader runs on each vertex
  • WPO output offsets vertex position
  • Happens on GPU (very fast)
  • Can use textures, math, time, etc.

Key Concept: WPO moves vertices, but doesn’t add new geometry. Limited by mesh vertex density.

WPO - Common Use Cases

Animation

  • Grass/foliage wind
  • Flag waving
  • Cloth simulation
  • Water waves
  • Breathing/idle motion

Deformation

  • Explosion shockwaves
  • Magic effects
  • Vertex displacement
  • Procedural animation
  • Interactive vegetation

Triplanar Mapping

Projection from 3 Axes (X, Y, Z)

Triplanar mapping projects textures from three orthogonal directions (X, Y, Z axes) and blends them based on surface normal, eliminating UV stretching.

The UV Stretching Problem:

  • Complex geometry has UV distortion
  • Cliffs and rocks stretch textures badly
  • Procedural meshes may lack good UVs
  • Solution: Project from world space instead

Key Concept: Sample texture 3 times (XY, XZ, YZ planes), blend based on surface normal direction

Triplanar - How It Works

OTOY Docs. (2025). Octane for ArchiCAD - Triplanar. [online] Available at: https://docs.otoy.com/ArchiCADH/ArchiCAD/Triplanar.htm [Accessed 6 Oct. 2025]. ‌

Algorithm:

  1. Sample texture 3 times:

    • X-axis projection (YZ plane)
    • Y-axis projection (XZ plane)
    • Z-axis projection (XY plane)

  2. Calculate blend weights:

    • Use world normal to determine which projection is most visible
    • Normalize weights so they sum to 1

  3. Blend results:

    • Multiply each sample by its weight
    • Add them together

Triplanar - UE5 Node

Using the Built-In Node

UE5 provides WorldAlignedTexture node that handles triplanar automatically:

Inputs:

  • Texture - The texture to project
  • World Position - Usually WorldPosition node
  • World Normal - Usually VertexNormalWS
  • Texture size - Scale of projection
  • Blend contrast - Transition sharpness

Output:

  • RGB - Blended color result

One node does all the work! Much cleaner than manual setup.

Triplanar - Key Benefits

Advantages

  • No UV stretching - Perfect projection
  • No UVs required - Works on any mesh
  • Seamless - No visible seams
  • World-aligned - Consistent across objects
  • Procedural-friendly - Great for generated meshes

Limitations

  • 3x texture samples - Performance cost
  • World-aligned only - Can’t rotate per-object
  • Tiling visible - Need good tiling textures
  • No unique detail - Same texture everywhere
  • Normal map complexity - Needs special handling

Triplanar + Vertex Painting

Combining Techniques for Best Results

Hybrid Approach:

  1. Base material uses standard UVs
  2. Triplanar detail layer for world-aligned effects
  3. Vertex paint to control triplanar blend

Use Case:

  • Cliff face with moss growing on it
  • Moss uses triplanar (world-aligned, grows “down”)
  • Vertex paint controls where moss appears
  • Base rock uses UVs for unique detail

Distance Fields Introduction

Volumetric Representation of Meshes

Epic Games Developer. (2025). 网格体距离场 | 虚幻引擎 4.27 文档 | Epic Developer Community. [online] Available at: https://dev.epicgames.com/documentation/zh-cn/unreal-engine/mesh-distance-fields?application_version=4.27 [Accessed 6 Oct. 2025]. ‌

Distance Fields are 3D volumetric data structures that store the distance from any point in space to the nearest surface.

How They Work:

  • Mesh is converted to volumetric grid
  • Each voxel stores distance to nearest surface
  • Positive = outside mesh
  • Negative = inside mesh
  • Zero = on surface

Distance Fields - Why Use Them?

Advantages Over Traditional Methods

Traditional Approach

  • Ray tracing for shadows (expensive)
  • Screen-space AO (limited range)
  • No proximity detection
  • Expensive collision queries

Distance Field Approach

  • Fast shadow approximation
  • Long-range AO
  • Cheap proximity queries
  • Efficient collision detection
  • Material effects based on distance

Key Benefit: Query “how far to nearest surface” is extremely fast with distance fields

Enabling Distance Fields

Project Setup

Project Settings:

  1. Edit → Project Settings
  2. Engine → Rendering
  3. Check Generate Mesh Distance Fields
  4. Restart editor
  5. Wait for distance fields to build

Per-Mesh Settings:

  • Most meshes auto-generate
  • Check Static Mesh Editor → Details → Distance Field
  • Adjust resolution if needed (higher = more accurate, more memory)
**Build Time:** Large projects may take hours to generate all distance fields. Plan accordingly!
