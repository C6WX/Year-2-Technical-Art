What Are Particle Systems?

The Basics: What is a Particle?

Particle systems are an efficient way to instantiate and manage large numbers of simple objects (called particles) or effects in a scene

Instead of manually spawning hundred or thousands of small objects, a particle system automates their creation and behavior using rules like:

  • Spawn Rate: How often new particles are created
  • Lifetime: How long each particle exists
  • Movement: How particles move (velocity, acceleration)
  • Appearance: How particles look (color, size, texture)
  • Particle systems are optimized to handle thousands of particle efficiently
  • They provide a high level of control and customization
  • We use them for effects like fire, smoke, rain, snow, explosions, magic spells, and more

What is Niagara?

Niagara is Unreal Engine 5‚Äôs tool for creating particle systems

Using Niagara we have two classes we can create, a Niagara System and a Niagara Emitter

  • Niagara is named after Niagara Falls in North America.

What is a Niagara Emitter?

A Niagara Emitter is a blueprint that defines the behavior of a single type of particle. It controls how particles are spawned, move, and die.

We can think of it as the blueprint for one type of particle system (e.g., sparks, smoke, embers)

Each Niagara Emitter controls:

  • Spawn rate and timing
  • Particle properties (Size, color, lifetime, velocity, etc).
  • Forces and behaviors (gravity, drag, collisions, attraction. etc)
  • Render type (sprite, mesh, ribbon, etc.)

What is a Niagara System?

A Niagara System is the container that holds one or more Niagara Emitters. It defines how those emitters interact and how the effect behaves as a whole.

Niagara systems are what we drag into the world and spawn in runtime. You can‚Äôt drag Niagara emitters directly into the world.

Niagara systems allow you to:

  • Combine multiple emitters to create complex effects
  • Control how emitters interact with each other
  • Define global properties and behaviors for the entire effect
  • Expose parameters for easy tuning in the editor or during gameplay

We can think of Niagara systems as the final effect, and the emitters as the building blocks that make up the effect.

How Particles work in Niagara Emitters

Every particle goes through these stages:

1. Spawn

  • The particle is created
  • It gets initial properties: position, color, size, direction
  • Example: A spark appears at the campfire

2. Life (Update)

  • The particle moves and changes
  • It follows rules: gravity pulls it down, wind pushes it sideways
  • Its appearance changes: it might get bigger, change color, or fade
  • Example: The spark rises up, glows orange, then dims

3. Death (Destroy)

  • The particle reaches the end of its lifetime
  • It disappears from the game
  • Memory is freed for new particles
  • Example: The spark fades to black and vanishes

Niagara Emitter Categories

Each Niagara Emitter is made up of a series of modules, grouped into categories that control different aspects of how the particle behaves.

Category: Emitter Properties

Controls Properties that are handled by the emitter.

  • Whether the particles spawned by the emitter are relative to the emitter origin (local space) or world space
  • Set randomness to be deterministic or not
  • Whether the particle simulation is handled by the CPU or GPU
  • Whether each particle spawned requires Persistent IDs

We can think of emitter properties as the class defaults for the emitter.

Category: Particle Spawn

Defines what happens when each particle is created

We can think of Particle Spawn as the Begin Play event of each particle.

  • Initial velocity and direction
  • Lifetime and initial size
  • Randomization and variation setup

Category: Particle Update

Defines what happens every frame for every living particle

We can think of Particle Update as the Tick event of each particle.

  • Movement and acceleration
  • Color and size over lifetime
  • Rotation, drag and forces
  • Collisions and custom behavior logic

Category: Render

Controls how particles are drawn in the scene.

  • Type of renderer (Sprite, mesh, ribbon, etc.)
  • Material and texture settings
  • Alignment, facing mode, and translucency

Category: Events

Controls how particles interact with each other and respond to events.

  • Sending and receiving events
  • Triggering actions based on conditions
  • Custom logic and behavior

Modules: Quick Essentials

Niagara effects are built by stacking modules. Combining and manipulating these modules allows you to create interesting effects. We can add modules to each category, but you will commonly add modules on

  • Particle spawn which will trigger whatever instruction is in the added modules when the particle spawns.

  • Particle update which will trigger whatever instruction is in the added modules each frame the particle is alive for.

Essential modules you‚Äôll use a lot:

1) Initialize Particle (in particle Spawn)

  • Sets initial Lifetime, Color, Size, Sprite Rotation, etc.
  • Use when: You need starting values and variation (ranges)

2) Spawn Rate / Spawn Burst (in Emitter Update)

  • What: Rate = continuous; Burst = instant one-shot
  • Use when: Fire (Rate), Explosion (Burst)
  • Tip: Lower lifetime reduces total particles at the same rate

3) Add Velocity (in Particle spawn)

  • What: Gives particles direction and speed
  • Use when: Upward fire, sparks radiating outward

4) Gravity Force (in Particle update)

  • What: Pulls particles downward (or upward with negative)
  • Use when: Sparks falling, smoke slowly rising (-)

5) Drag (in Particle Update)

  • What: Slows particles over time (air resistance)
  • Use when: Smoothly damping fast motion (sparks, debris)

6) Curl Noise Force (in Particle Update)

  • What: Adds natural, turbulent motion
  • Use when: Fire flicker, smoke swirl, magic energy

7) Scale Color (in Particle Update)

  • What: Changes color and alpha over life
  • Use when: Fade to transparent, color shifts over time

8) Scale Sprite Size (in Particle Update)

  • What: Grow/shrink over life
  • Use when: Smoke expands, fire licks stretch then shrink

Curves & Normalized Age

Curves let you control a value over time using a small graph.

  • X-axis = time (we usually use Normalized Age: 0 at birth ‚Üí 1 at death)
  • Y-axis = the value you want to change (size, alpha, color channel, etc.)

Why Normalized Age?

  • Independent of actual lifetime (works for 0.5s or 3s particles)
  • Reusable: You can change lifetime and the curve still fits perfectly

Fade Out with a Curve (Alpha)

  1. Add ‚ÄúScale Color‚Äù in Particle Update
  2. Click the input to switch to a Curve (use the curve icon / dropdown)
  3. Set X-axis to Normalized Age (0 ‚Üí 1)
  4. Add keys: at 0 ‚Üí 1.0 (fully visible), at 1 ‚Üí 0.0 (invisible)
    Result: All particles fade out at the end of life

Size Over Life with a Curve

  1. Add ‚ÄúScale Sprite Size‚Äù in Particle Update
  2. Switch its input to a Curve
  3. Keys: 0 ‚Üí 0.6, 0.3 ‚Üí 1.2, 1.0 ‚Üí 0.0 (grow then shrink)
    Result: Natural growth and shrink behavior

Color Over Life with a Color Curve

  1. In ‚ÄúScale Color‚Äù, switch to a color curve
  2. Keys: at 0 ‚Üí orange, 0.5 ‚Üí red, 1 ‚Üí dark red/black
    Result: Fire shifts from bright to dark naturally

Attributes You‚Äôll Use

  • Age (seconds): how long a particle has lived
  • Lifetime (seconds): how long it will live total
  • Normalized Age (0‚Üí1): Age / Lifetime
  • Position, Velocity, Color, Sprite Size, Sprite Rotation, Mass, Drag, etc.

Exposing Parameters (User Parameters)

User Parameters let designers and Blueprints control your effect without opening Niagara.

Create a User Parameter

  1. Open your Niagara System
  2. In the Parameters panel, click + to add a parameter
  3. Choose a type: Float, Integer, Bool, Vector, Linear Color, etc.
  4. Name clearly, e.g., Fire_Intensity (Float), Fire_Color (Linear Color)

Bind a User Parameter to a Module Input

  1. Find an input you want to control (e.g., Spawn Rate, Color)
  2. Click the input‚Äôs dropdown and choose Bind to Parameter ‚Üí User ‚Üí Fire_Intensity (or ‚ÄúMake ‚Üí User Parameter‚Äù)
  3. The module now reads from your User parameter

Example: Control Spawn Rate from a Parameter

  • Add Float parameter: Fire_Intensity (default 1.0)
  • Bind Spawn Rate = 50 * Fire_Intensity
  • Result: Setting Fire_Intensity to 2.0 doubles particle count

Example: Control Color from a Parameter

  • Add Linear Color parameter: Fire_Color
  • Bind Initialize Particle ‚Üí Color = Fire_Color
  • Result: You can recolor the effect at runtime

CPU vs GPU Simulation

Two ways to simulate particles:

CPU (CPUSim)

  • Great for: Complex logic, events, collisions, small/medium counts
  • Typical counts: up to a few thousand
  • Pros: Most modules supported; easy events/collision
  • Cons: Slower for huge counts

GPU (GPUComputeSim)

  • Great for: Massive counts (rain, snow, sand, sparks)
  • Typical counts: tens/hundreds of thousands
  • Pros: Very fast for simple behaviors
  • Cons: Many CPU-only features aren‚Äôt supported (some collisions, event handlers)

How to Switch

  1. Select an emitter ‚Üí Emitter Properties (top of stack)
  2. Find Sim Target
  3. Choose CPUSim or GPUComputeSim

If you switch to GPU and see warnings, some modules aren‚Äôt supported on GPU. Replace them with GPU-compatible alternatives or simplify behavior.

When to Use Which

  • Use CPU for: gameplay-critical effects, collisions, events, complex logic
  • Use GPU for: background/ambient effects and very high particle counts

Blueprint Integration (Controlling Niagara from Gameplay)

Option A: Add a Niagara Component to an Actor

  1. In your Blueprint Actor, add a Niagara System component
  2. Set the Niagara System asset
  3. Toggle Auto Activate if you want it to play on BeginPlay

Option B: Spawn at Runtime

  • Use node: Spawn System at Location (Niagara)
  • Or: Spawn System Attached (Niagara) to follow a socket/bone
  • Set Auto Destroy to clean up automatically after finish

Setting User Parameters from Blueprints

  • Get Niagara Component ‚Üí Set Niagara Variable (Float)
    • Variable Name: User.Fire_Intensity
    • Value: 1.5
  • Get Niagara Component ‚Üí Set Niagara Variable (Linear Color)
    • Variable Name: User.Fire_Color
    • Value: color from gameplay

Parameter names in Blueprint should include the User. prefix, e.g., User.Fire_Color

Simple Example Flow

  1. Player presses ‚ÄúFire‚Äù ‚Üí Spawn System at Location (NS_Fireball)
  2. Immediately Set Niagara Variable (Float) User.Speed = ProjectileSpeed
  3. While charging, update User.Intensity over time
  4. On impact, spawn a different Niagara System for the explosion
